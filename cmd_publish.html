publish

The publish command generates static files for your site from the current state of your files in your source and sass directories. It then deploys your site to your remote server. If you have initialized your site directory with Git, this command will also automatically git commit your file changes, using a timestamp as the commit message.

The default command publishes to your GitHub Pages site or rsync managed site as per your specifications during your Octopress install. You have the option to specify deployment to Heroku.

Usage

oc publish

Arguments

None

Options

Option Description
--heroku deploy to Heroku
